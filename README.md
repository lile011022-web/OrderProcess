# 球星卡采购接单对账系统

版本：v1.3.0

这是面向球星卡采购、接单、包裹入库、费用核算、付款确认、异常处理和对账管理的业务系统。v1.3.0 已加入 SQLite 持久化后端、token 登录、角色校验、审计日志、凭证上传、登录页改密、生产清除测试数据、基础安全加固，以及采购任务、接单、买手回填、付款、仓库确认、异常处理、客户资料和报表导出的真实写入闭环。

仓库名称：OrderProcess。用途：个人记账、采购流程管理和球星卡供销对账。

## 技术栈

- React
- Vite
- TypeScript
- Tailwind CSS

## 本地运行

```bash
npm install
npm run dev
```

生产构建验证：

```bash
npm run build
```

后端 API：

```bash
npm run backend:dev
npm run backend:check
npm run backend:clear-business-data
```

后端默认监听 `http://127.0.0.1:7301`，默认 SQLite 数据文件在 `data/order-process.sqlite`，部署说明见 [docs/backend-deployment.md](docs/backend-deployment.md)。

阿里云服务器 `47.242.190.166` 部署说明见 [docs/aliyun-deployment.md](docs/aliyun-deployment.md)。

## v1.2 功能范围

- 四种角色登录与不同菜单权限。
- 管理员采购任务、买手接单、采购回填、付款确认、包裹管理、仓库收货、成本核算、报表和基础资料维护。
- 客户独立页面：我的采购任务、我的商品资料、我的仓库地址。
- 运单号集中识别并跳转 UPS、FedEx、USPS 官网查询。
- 仓库操作费集中计算：5 美元 / 包裹，0.5 美元 / 张照片，固定汇率 6.8。
- 付款后先进入“已付待确认金额”，仓库确认收到后才转为“实际入库成本”。
- 异常包裹独立记录退款或下次抵扣，不混入正常入库成本。
- Node.js + SQLite 后端，覆盖登录、菜单、采购、包裹、异常、费用、对账、上传和审计接口。
- 前端登录、采购任务、包裹、异常、成本核算、仓库收货、客户资料、买手付款等核心页面已接后端 API。
- 生产环境默认不写入业务测试数据，如需演示数据必须显式设置 `SEED_DEMO_DATA=true`。
- 登录和改密接口增加限流，后端与 Nginx 增加基础安全响应头。
- 登录首页提供修改密码入口，不再在页面展示默认测试密码。
- v1.3 起，发布采购任务、接单、买手回填、管理员付款、仓库确认、异常处理、商品/仓库资料提交和审核、CSV 报表导出均会真实调用后端并写入 SQLite。

## 验收建议

- 打开 `/login`，分别使用四种角色登录，确认默认入口和菜单不同。
- 未登录访问 `/dashboard`、`/packages` 等后台路径，确认会回到 `/login`。
- 使用管理员查看包裹列表、异常包裹、仓库操作费和成本核算。
- 使用买手查看任务大厅、我的接单和买手收款状态。
- 使用仓库查看待确认包裹、已收货包裹和超时待核查。
- 使用客户查看我的商品资料和我的仓库地址，确认只显示客户自己的数据。
- 点击 UPS / FedEx / USPS 运单号，确认使用集中规则打开官网查询。
- 在登录页点击“修改密码”，确认弱密码被拒绝，强密码可修改，旧密码失效。
- 生产部署后执行 `npm run backend:clear-business-data` 清除业务测试数据，再确认各业务列表为空。
- 使用管理员发布采购任务，再使用买手接单和回填，确认包裹列表出现对应包裹。
- 使用管理员付款后，确认包裹金额进入“已付待确认”；使用仓库确认收到后，确认金额转为“实际入库成本”。
- 使用客户提交商品/仓库地址，再用管理员在基础资料中审核启用。
- 在报表页点击导出 CSV，确认浏览器下载报表文件。

## 项目结构

```text
src/
  components/      通用表格、筛选器、弹窗、状态标签、统计卡片等
  data/            集中 mock 数据与快递配置
  layouts/         登录后后台布局
  pages/           各业务页面
  routes/          路由与登录保护
  utils/           权限、登录态、费用、格式化等集中规则
  types.ts         核心业务类型
server/            Node.js SQLite API 后端
```
