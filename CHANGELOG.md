# Changelog

## v1.3.3 - 2026-07-06

- 买手采购回填新增真实凭证上传，支持采购凭证、订单截图和运单截图随回填记录保存到后端上传表。
- 买手“我的接单”和“买手付款”改为读取真实买手回填记录，买手端只能查看和上传凭证，管理员端保留审核、付款和驳回操作。
- 客户采购任务新增真实结款状态，客户可确认结款，管理员采购任务列表可查看客户结款金额和时间。
- 后端权限放开已启用商品库和仓库地址的跨角色读取，客户发布任务和买手回填可直接使用管理员维护的基础资料。
- 完成四角色核心链路压力测试：客户发布任务、买手接单与回填、管理员审核付款、仓库确认入库、异常处理、负数费用拦截、非法上传拦截和并发创建任务。
- 清理本地业务测试数据后保留系统基础资料，当前任务、买手回填、包裹和异常记录均为 0。
- 将 `最新地址.docx` 中的 7 个美国仓库地址升级为系统基础资料，不再依赖演示数据开关。
- 后端启动时自动保证 7 个最新地址存在；生产环境 `SEED_DEMO_DATA=false` 也会显示这些仓库地址。
- 清除测试数据后会自动恢复最新地址，避免“仓库地址”页面变成 0 条记录。
- `npm run backend:import-latest-addresses` 改为幂等 upsert，可作为服务器旧数据库的兜底补导命令。
- 商品库“新增商品”改为真实弹窗表单，“编辑”改为真实编辑表单；修复浏览器 PATCH/DELETE 预检被 CORS 拦截导致编辑、审核启用、停用失败的问题。
- 商品库新增/编辑支持上传商品照片；发布采购任务可从商品库选择已启用商品，并自动带出商品名称、规格、参考价和照片。
- 商品照片上传后，商品编辑弹窗改为固定高度滚动布局，底部保存按钮始终可点击；发布采购任务的收货仓库、收货人和采购要求改为非必填。
- 新增 13 个 2025/26 篮球卡盒系统商品资料，包含 DA 商品链接、商品照片、品牌、规格和参考价；全新数据库、清除测试数据和 `npm run backend:import-latest-products` 都会保留/补导这些商品，手动删除后普通重启不会偷偷恢复。
- 商品库操作列新增“删除”按钮，删除前二次确认，删除后会真实调用后端 DELETE 接口并从商品库和发布采购任务商品下拉中移除。
- 优化商品库表格布局，将商品名称、编码、分类、品牌合并为“商品资料”，来源与链接合并展示，右侧操作区改为紧凑 2x2 按钮组，避免按钮竖向堆叠挤压。

## v1.3.2 - 2026-07-06

- 导入 `最新地址.docx` 中的 7 个最新美国仓库地址，前端默认资料和后端 SQLite 初始化数据同步可见。
- 新增 `npm run backend:import-latest-addresses`，用于在已有服务器数据库中补导最新地址。
- 修复多个页面按钮点击无反应：客户任务进度/复制再发、仓库超时核查、仓库费用明细/结算、成本明细/分摊、买手流水查看、仓库照片保存、ETA 提醒处理等。
- 仓库待确认页的“搜索包裹”和“上传包裹照片”改为真实反馈/弹窗动作。
- 表格分页按钮在当前单页模式下显示为禁用态，避免用户误认为点击失效。
- 首页趋势图“本年”按钮增加即时反馈。

## v1.3.1 - 2026-07-06

- 修复采购任务和买手回填的数字输入，输入 `020`、`02250` 等会立即规范为 `20`、`2250`。
- 买手回填的税费、美国境内运费、买手服务费默认改为 `0`。
- 买手回填的采购平台默认改为 `whatnot`，并随回填记录保存到后端。
- 买手填写的收货人姓名优先级高于管理员维护的仓库收货人；生成包裹时优先使用买手填写值，未填写时才回退到仓库联系人。

## v1.3.0 - 2026-07-04

- 将核心原型按钮接入真实后端写入：发布采购任务、保存草稿、任务接单、任务复制、暂停、完成和删除。
- 买手采购回填可真实保存到 SQLite，并自动生成待处理包裹。
- 管理员审核/驳回买手回填、记录付款，付款金额进入包裹“已付待确认”。
- 仓库可更新 ETA、登记未收到异常、确认收到包裹，确认后待确认金额转为实际入库成本。
- 包裹列表支持真实标记异常，异常包裹处理结果可保存并同步包裹异常金额。
- 客户商品资料、客户仓库地址可真实提交；管理员商品库和仓库地址可新增、审核启用、停用。
- 新增通用业务记录 API：创建、更新、删除统一写审计日志。
- 报表页和首页支持导出 CSV。
- 修正列表过滤逻辑，避免客户采购任务因 `owner/requester` 字段差异被误过滤。

## v1.2.1 - 2026-07-03

- 生产环境默认关闭业务测试数据写入，仅保留四个初始账号用于首次登录。
- 新增 `npm run backend:clear-business-data`，可清除采购、包裹、对账、基础资料和上传凭证测试数据。
- 登录首页新增“修改密码”界面，支持四种角色通过旧密码修改为强密码。
- 新密码增加强度校验：至少 10 位，必须包含大小写字母、数字和特殊符号，并拒绝常见弱密码片段。
- 登录和改密接口增加基础限流，降低暴力破解风险。
- 后端和 Nginx 增加安全响应头，systemd 服务增加基础沙箱限制。
- 部署脚本自动写入 `SEED_DEMO_DATA=false` 和认证限流配置，避免重新部署时误开演示数据。

## v1.2.0 - 2026-07-03

- 后端从内存 mock API 升级为 SQLite 持久化 API。
- 新增阿里云 `47.242.190.166` 部署文档、systemd 服务模板、Nginx 反代模板和部署脚本。
- 前端 API 地址支持生产同域反代，部署时可使用 `VITE_API_BASE_URL=""`。
- 登录改为服务端密码哈希校验与 HMAC token。
- 写操作增加角色校验和审计日志。
- 增加图片/PDF 凭证上传接口和本地落盘记录。
- 仓库费用接口增加整数、非负校验，拒绝负数和小数。
- 前端登录改为调用后端 API，核心业务页面逐步接入后端列表接口。
- 仓库确认收货前端动作已接后端，待确认金额会转入实际入库成本并持久化。
