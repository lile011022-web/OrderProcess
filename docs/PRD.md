# 球星卡采购接单对账系统 PRD

版本：v0.2
更新时间：2026-07-02
阶段：v1.2 核心业务可用化

补充：v1.2 已新增 SQLite 持久化后端、token 登录、角色校验、审计日志、凭证上传和核心页面 API 接入。部分页面仍保留 mock fallback，用于后端不可用时展示基础界面。

## 1. 产品目标

本系统用于管理球星卡采购任务、买手接单采购、采购回填、包裹物流、仓库收货、付款确认、异常处理、基础资料和对账统计。v1.2 已具备基础持久化后端，但暂不调用真实物流接口。

## 2. 本次修订结论

- 不再提供单独的“费用规则”设置入口，避免用户误以为第一版需要配置费用规则。
- 运单号由系统自动识别 UPS、FedEx、USPS，买手不再手动选择快递公司。
- 买手不再填写预计到达时间，预计送达时间由仓库端在待确认包裹中填写或更新。
- 收货仓库由管理员维护，买手回填时通过下拉框选择已启用仓库。
- 买手需要填写收货人姓名。
- 管理员付款、买手收款均按采购数量处理，支持全选或按部分数量处理。
- 仓库照片不再作为单独菜单，改为在已收货包裹或超时核查的对应单号下上传。
- 超时待核查需要支持上传官网截图、包裹照片或异常反馈照片。
- 商品库只管理商品基础资料，由管理员维护或审核客户上传内容。
- 仓库地址由管理员和客户提交维护，地址编辑采用美国地址字段。
- 客户端页面必须和管理员端分隔：客户只看自己的商品资料和仓库地址提交，管理员看全量基础资料并负责审核。
- 交互中纯图标按钮需要提供明确悬停说明，当前选中菜单需要与 hover 状态明显区分。

## 3. 角色与权限

系统支持四种角色：

- 管理员：发布采购任务，管理买手、包裹、基础资料、财务对账和报表。
- 买手：查看任务大厅，接单，回填采购结果，查看自己的收款状态。
- 仓库：处理待确认包裹、已收货包裹、超时待核查和仓库操作费确认。
- 客户：提交和维护自己的商品资料、美国仓库地址，并查看审核状态。

权限规则必须集中维护，页面不得散落硬编码角色判断。

## 4. 核心业务流程

1. 管理员发布采购任务，填写商品、数量、目标价、采购要求和收货信息。
2. 买手在任务大厅接单并完成采购。
3. 买手回填实际采购数量、单价、费用、平台订单号、运单号、收货仓库和收货人。
4. 系统根据运单号识别快递公司，并提供官网查询跳转。
5. 管理员审核买手回填记录。
6. 财务按采购数量付款，可全选，也可选择部分数量付款；付款后金额进入“已付待确认”。
7. 仓库在待确认包裹中填写或更新预计送达时间，并按运单号确认收货。
8. 仓库确认实际收到后，相关金额才转为实际入库成本。
9. 已收货包裹的照片在对应运单号详情中上传和查看。
10. 超时未收货的包裹进入核查流程，仓库可上传官网截图、包裹照片或异常照片。
11. 异常金额单独记录，支持退款或下次抵扣，不混入正常入库成本。

## 5. 页面需求

### 5.1 买手采购回填

- 买手填写接单编号、商品名称、采购数量、采购单价、税费、美国境内运费、买手服务费、采购平台、平台订单号和采购备注。
- 买手填写运单号后，系统自动显示识别出的快递公司。
- 预计到达时间不在买手端填写。
- 收货仓库从管理员维护的启用仓库中选择。
- 收货人由买手填写姓名。
- 支持采购截图、订单截图和运单截图上传入口。

### 5.2 买手收款 / 管理员付款

- 买手端展示为“买手收款”，管理员端展示为“买手付款”。
- 列表展示采购数量、结算金额、收款/付款数量、运单号、系统识别快递、仓库预计到达、审核状态和收款/付款状态。
- 收款/付款处理支持按采购数量计算金额。
- 支持“全选”和“本次数量”输入。
- 付款确认后只进入“已付待确认”，不得直接进入实际入库成本。

### 5.3 仓库待确认包裹

- 仓库可以扫描或输入运单号搜索包裹。
- 列表展示运单号、快递公司、预计到达、是否超时、商品摘要、应收数量、买手、仓库和状态。
- 仓库可填写或更新预计送达时间，并记录官网查询或仓库备注。
- 仓库确认收到时填写实际收到时间、实收数量、包裹照片、商品照片、异常照片和收货备注。

### 5.4 已收货包裹

- 已收货包裹必须是专用业务页面，不使用通用占位表格。
- 每个已收货运单号提供上传照片入口。
- 照片包括外箱照片、商品照片和异常补充照片。
- 已有照片按对应运单号展示。

### 5.5 超时待核查

- 超时包裹不直接生成异常金额。
- 仓库可选择继续等待、已收到、未收到或物流异常。
- 支持填写官网查询结果备注。
- 支持上传官网截图、包裹照片或异常照片。
- 支持提醒管理员。

### 5.6 商品库

- 商品库只维护商品基础资料，不展示采购任务式的通用表格。
- 字段包括商品编码、商品名称、分类、品牌、基础规格、资料来源、参考价和状态。
- 管理员可新增、编辑、审核客户上传资料。

### 5.7 仓库地址

- 仓库地址由管理员维护或审核客户提交。
- 买手回填时只能选择已启用仓库。
- 地址编辑采用美国地址字段：Address Line 1、Address Line 2、City、State、ZIP Code、Country。
- 列表展示仓库名称、维护方、收货人、电话、美国地址和状态。

### 5.8 客户端页面

- 客户登录后进入独立的客户资料菜单，不进入管理员基础资料菜单。
- “我的商品资料”只展示当前客户提交的商品，并支持提交商品资料、编辑和重新提交审核。
- “我的仓库地址”只展示当前客户提交的地址，并支持按美国地址字段提交、编辑和重新提交审核。
- 客户页面不展示管理员审核、设为启用、全量资料维护等管理动作。
- 管理员仍在“商品库”和“仓库地址”页面查看全量资料并执行审核。

## 6. 集中规则

### 6.1 运单识别

系统集中维护 UPS、FedEx、USPS 的识别规则和官网查询链接。页面不得自行拼接查询链接。未知运单号展示“待识别”，不跳转错误官网。

### 6.2 付款与入库成本

买手或管理员付款后，金额只能进入“已付待确认”。仓库确认实际收到后，才能转为“实际入库成本”。未经仓库确认，不允许直接计入实际入库成本。

### 6.3 异常处理

异常需记录原因、责任方、涉及金额、处理方式、状态、备注和凭证。异常金额需要在对账中清晰区分，不能混入正常入库成本。

## 7. 原型验收标准

- 未登录访问后台页面跳转到 `/login`。
- 四种角色可模拟登录，登录后菜单不同。
- 客户可使用 `customer / 123456` 登录，登录后只能看到客户资料菜单。
- 客户页面与管理员基础资料页面分隔，客户不看到全量商品库或全量仓库地址。
- 买手菜单不会出现同一路径导致两个入口同时高亮。
- 买手回填只填运单号，系统显示快递识别结果。
- 买手回填中的收货仓库为下拉选择，收货人为买手填写。
- 买手收款/管理员付款支持按采购数量全选或部分处理。
- 仓库待确认中可以填写预计送达时间。
- 已收货包裹中可以按对应单号上传照片。
- 超时待核查中可以上传官网截图和异常反馈照片。
- 商品库展示商品基础资料。
- 仓库地址页面展示和编辑美国地址字段。
- 纯图标按钮具备悬停说明。
